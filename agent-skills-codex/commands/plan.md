---
description: Break work into small verifiable tasks with acceptance criteria and dependency ordering
---

Use skill: `planning-and-task-breakdown`.

Read the existing spec (`SPEC.md` or equivalent) and relevant codebase sections. Then:

1. Enter planning mode (read-only)
2. Identify dependency graph between components
3. Slice work vertically (complete paths, not horizontal layers)
4. Write tasks with acceptance criteria and verification steps
5. Add checkpoints between phases
6. Present plan for human review

Save plan to `tasks/plan.md` and task list to `tasks/todo.md`.
