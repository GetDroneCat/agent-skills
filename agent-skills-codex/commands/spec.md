---
description: Start spec-driven development — write a structured specification before writing code
---

Use skill: `spec-driven-development`.

Begin by understanding what the user wants to build. Ask clarifying questions about:
1. Objective and target users
2. Core features and acceptance criteria
3. Tech stack preferences and constraints
4. Known boundaries (always do, ask first, never do)

Generate a structured spec covering objective, commands, structure, code style, testing, and boundaries.

Save as `SPEC.md` in project root and confirm with the user before implementation.
